# Завдання за модулем 1 
## Опис основних сутностей
Для виконання цього завдання була вибрана предметна область комп'ютерної
техніки, а саме дві сутності - ноутбук та виробник. У сутності "Ноутбук"
є один виробник, а також ціна, модель, місткість оперативної пам'яті, 
а також перелік доступних портів. Останній атрибут має кілька значень, 
які зберігаються в одному рядку і записані через кому.

Основну роботу програми виконують два класи - ParseFromJson та OutputToXml.
У першому визначено метод, який шукає необхідні атрибути та записує їхню
кількість у спеціальній мапі, яка дозволяє паралельні операції. Основний
метод створює віртуальні потоки для кожного файлу в поданому розташуванні.
Потоки виконують метод для обробки файлу та записують результат у мапі 
ConcurrentMap, після чого конвертує цю мапу на звичайну та повертає 
результат. У класі застосовано семафор, щоб обмежити максимальну кількість
одночасних доступів до файлів. Цей захід вжито для запобігання помилки
виконання програми при дуже великих кількостях файлів.

Основний метод класу OutputToXml приймає звичайну мапу, після чого обробляє
кожен запис у мапі, перетворюючи його на об'єкт StatisticsItem та створюючи
список цих об'єктів. Потім із цього списку створюється об'єкт типу
StatisticsReport. Наостанок створюється файл для запису статистики, після
чого за допомогою методу бібліотеки jackson databind дані з об'єкта
StatisticReport записуються в файлі XML.

Використання віртуальних потоків разом із семафором забезпечує вищу
швидкість та менше використання ресурсів у порівнянні з потоками ОС, адже
віртуальні потоки - це звичайні об'єкти Java, які "прикріплюються" до
потоків ОС, з чого виникає, що їх можна створювати більше. Також
віртуальні потоки вимагають менше пам'яті. Проте вони корисні тільки у
випадку, коли потрібно обробити велику кількість простих завдань.
